# Autodesk CFD 2027 操作手順（GUI・クリック単位／**非定常解析**）

Case A / B / C を**同一設定**で解析し，推力 \(F_z\) を読み出すための Autodesk CFD 2027
（日本語UI）のクリック単位の手順．**授業資料 `Exercise05.pdf`（回転翼の非定常計算）に準拠**する．
本書は **A/B/C 統一 CFD の設定・操作・結果抽出・自動化メモを集約した単一の手順書**である
（旧 `scripts/cfd_setup.md` は本書へ統合した）．形状は [caseA.md](caseA.md) / [caseB.md](caseB.md) /
[caseC.md](caseC.md) を参照．

> **最重要（前回の失敗の原因）**：本問題は **非定常解析（Transient）**である．
> 解析モードを「非定常解析」にし，**時間刻み幅・終了時刻・実行する時間ステップ数**を必ず設定する．
> これらが未設定（実行ステップ 1〜0）だと**一瞬で「完了」表示が出るが計算されていない**．
>
> **力の抽出は壁面計算機ではなく CSV ファイル**から行う（資料に「壁面計算はできないようだ」と明記）．
> 出力 `シナリオ1_*****.csv` の **Hydraulic ForceZ（dyne）**を最終1回転で平均して推力とする．

> **比較の鉄則**：3 ケースで変えるのは**読み込む形状ファイルだけ**．回転数・境界条件・メッシュ方針・
> ソルバ・乱流モデル・時間刻みはすべて同一．Z 軸まわりの回転なので**回転中心は 3 ケースとも
> `(0,0,0)`**（Z 軸上のどの点でも Mz・回転は不変）．

## 0. 前提と本件の数値

- 入力形状：`geometry/<case>/<case>_cfd.step`（**プロペラ＋外箱の入れ子2ソリッド**）．
  無ければ Windows/Inventor で `python scripts/cfd_domain_inventor.py` で再生成．
  - ★ **中間の回転円筒は廃止**した（A/B/C 共通）．回転はプロペラ固体に与え，CFD が周囲空気を回すので
    円筒流体は不要（OpenFOAM-MRF の名残だった）．近傍メッシュを細かくしたいときは CFD 側で
    **シリンダーのメッシュ細分化“領域”**（ジオメトリ実体ではない）を追加すればよい（§5）．
- 単位：STEP は mm だが CFD は cm 表示（「ジオメトリ (cm)」）．長さ欄は cm 注意．

**ハードウェア方針（GPU について）**

- **Autodesk CFD のソルバは CPU 専用．GPU 高速化は非対応**（公式：CPU のみ）．本 PC の GPU
  （VRAM 6 GB）は**描画・ポスト処理のみ**に効き，解析時間短縮には使えない（⇒ VRAM は制約にならない）．
- 速度を決めるのは **CPU コア数（ソルバは 2ⁿ＝2/4/8/16… を利用）と RAM**（目安 **約 2 GB/100 万要素**）．
  実行設定でコア数を **2ⁿ の最大**に．
- 参考：Autodesk, *Hardware Recommendation for Autodesk CFD*（CPU-only solver, 2ⁿ cores, ~2 GB/1M）．

**計算領域と形状（mm, Z＝回転軸＝推力方向）**

`<case>_cfd.step` は [scripts/cfd_domain_inventor.py](../scripts/cfd_domain_inventor.py) が
**プロペラ＋外箱の入れ子 2 ソリッド**で出力する．外箱寸法は全ケース共通，中心 z のみ形状に追従：

| ケース | 掃引直径 D | 形状 z 範囲 | 中心 z | 外箱（X,Y＝±180 / Z＝中心±300） |
|---|---|---|---|---|
| A | 96.0 mm | −10…+10 | 0 | Z `−300…+300` |
| B | 90.8 mm | −8…+8 | 0 | Z `−300…+300` |
| C | 98.0 mm | 0…+58 | +29 | Z `−271…+329` |

**回転数と時間刻み（本件は 100 rpm。授業資料の 100 rps＝628 rad/s ではない）**

| 量 | 値 | 備考 |
|---|---|---|
| 角速度 ω | **10.472 rad/s** | ＝100 rpm＝100×2π/60．モーション編集の「一定角速度」に入力 |
| 1 回転の時間 | 0.6 s | ＝60/100 |
| 時間刻み幅 dt | **0.003 s** | 1 回転を 200 分割（**1.8 deg/step**＝資料と同じ安定寄り） |
| 終了時刻（Stop Time） | **−1** | 時間では止めずステップ数で止める（§6・0 ステップ即終了の対策） |
| 実行する時間ステップ数 | **400**（結果用） | ＝0.003×400＝1.2 s＝**2 回転**．安定テストは **200**（＝1 回転）で可 |
| 次から継続 | **t0（最初）** | 前回の最終ステップから継続しない（§6 トラブルシュート） |
| 保存間隔（結果） | **0.03 s** | 1 回転(0.6 s)に 20 データ |
| **内部反復係数** | **3** | 層流＋これで安定．まだ落ちるなら 4〜5（§6 発散対策） |
| **流れ** | **圧縮性流体** | 安定化（資料の助言）．全温度 ≈ **300 K（26.85℃）**，伝熱オフ |

> 安定性メモ：当初の **非圧縮・dt0.006(3.6°/step)・内部反復1〜2** は回転翼で発散した（SST で step 66，
> 層流で step 167）．発散の主因は**1 ステップの回転量が大きすぎる**こと．対策の決定版は
> **dt 0.003（1.8°/step）＋圧縮性流体＋内部反復 3**（資料も「運動を伴う場合は圧縮性が安定」と明記）．
> **3 ケースで同一**．ステップは安定テスト 200（1 回転）→ 結果は 400（2 回転）に延ばす．

## 1. 形状の取り込み（スタディ作成）

1. CFD 起動 →「開始および学習」または「設計の追加/更新」から新規スタディ．
   （Inventor アセンブリがある場合は Simulation タブの Active model から起動してもよい）
2. ジオメトリに `geometry\caseA\caseA_cfd.step` を取り込む．
3. **2 つの入れ子ボリューム**ができる（「材料」配下に 2 パーツ）：
   - `...ソリッド2_U_ソリッド1`（内側）＝**プロペラ**（固体．回転運動を与える対象）
   - `...ソリッド2`（外側殻）＝**静止流体（外箱）**
   - 見分けは左ツリーでクリック→3D ビューのハイライトで（翼＝プロペラ，最大＝外箱）．
4. **半透明表示**にして，外箱の中にプロペラが収まっていることを確認する．
   プロペラの肉厚が薄いと警告が出るが**いったん無視**してよい（資料 2.1）．

## 2. 材料（リボン「セットアップ」→「材料」）

| ボリューム | 材料 |
|---|---|
| プロペラ（内側） | 任意の**固体**（例：ABS．構造は無視するので何でも可．**§4 で回転運動を付与**） |
| 静止流体（外箱殻） | **空気（Air）** |

## 3. 境界条件（リボン「セットアップ」→「境界条件」）

本プロジェクトは**ホバリング（純揚力）**想定なので，**外箱の全外面を圧力 0 Pa** とする（開放）．

1. リボン「選択」を**サーフェス**にし，**外箱の外側6面**を選択．
2.「境界条件」→ **圧力（ゲージ圧）＝ 0 Pa**．
3. プロペラ壁は既定の **no-slip 壁**（指定不要．資料 2.1「指定しない境界は壁条件」）．

> 参考：授業資料は前進飛行（風洞）なので入口速度 4 m/s・側面 x=4 m/s・出口 P=0 にしている．
> 本件は前進速度を与えないホバリングなので**全面 0 Pa**でよい（前進飛行を見たい場合のみ資料に倣う）．
> 機体ボディは作図していないので資料 2.1 の機体サーフェス速度設定（=0）は不要．

## 4. プロペラのモーション（リボン「セットアップ」→「モーション」／資料 2.2）

> **回転運動は流体には適用できない．プロペラ（ソリッド）に適用する．** Autodesk CFD は
> 「ソリッドに回転運動を与える → 周囲の流体を非定常で回す」方式（資料 2.2 もプロペラを直接選択）．
> これにより回転円筒流体は不要（§0 で廃止済み）．

1.「モーション」を選択し，画面上で**プロペラ本体（内側ソリッド `...ソリッド2_U_ソリッド1`）**を
   **ボリューム選択でクリック**して「編集」．
   - ※ この時だけ外箱を突き抜けて**直接プロペラをクリックできる**（資料 2.2）．
   - 左の「材料」ツリーから内側パーツを選んでもよい（3D ビューで翼形状がハイライトされる方）．
2. モーションウインドウで：
   - 種類：**回転運動**
   - 回転軸：**z（0,0,1）**
   - 回転の中心：**（0,0,0）**
   - 初期位置：**0 deg**
3.「編集」→ 変化方法「**一定角速度**」→ 角速度 **`10.472`** rad/s →「適用」．
   - ⚠️ 授業資料の `628` は 100 rps 用．**本件は 100 rpm なので 10.472 rad/s**．
4.「プレビュー」で回転方向を確認．

## 5. メッシング（リボン「セットアップ」→「メッシュサイズ」／資料 2.3）

1. いったん**自動で「メッシュを生成」**．
2. **外箱（遠方場）は粗く保つ — これが要素数・速度を左右する**
   外箱は ±180×±180×±600 mm と大きい．**プロペラ表面の細メッシュが外箱へ伸びて増殖**すると流体要素が
   爆発する（実測：薄翼面を細かくしたら**流体だけで 5.1 M**＝合計 5.5 M に膨張）．対策：
   - 外箱ボリュームを選び「**編集**」→ サイズ調整を**粗め（0.5〜1.0）**にして，遠方場を粗く固定．
   - 細かくするのは**プロペラ近傍だけ**（下の項4 の薄翼面細分化，必要なら項3 のシリンダー領域）に限定する．
   - 目安：**合計要素 ≈ 1〜2 M**（流体が数 M に膨らんでいたら外箱が細かすぎ → 粗くする）．
   - ★ 外箱のサイズ調整値も **3 ケースで同一**にする．
3.（任意）**プロペラ近傍を細かくしたい場合**のみ，**細分化“領域”**を追加する（ジオメトリ実体は不要．
   円筒を廃止したのでメッシュ細分化は CFD 側の「領域」で行う）：
   - **細分化領域（シリンダー）**：プロペラを囲む円筒範囲を「追加」．領域タイプ「**シリンダー領域**」，
     **Y 角度 90°**（Z 回転に合わせる），局所メッシュ ≈ **0.25 cm**（低スペック PC は 0.5 cm）．
   - これは必須ではない（下の薄翼面細分化だけでも可）．使うなら **3 ケースで同一設定**にする．
4. **薄翼対策（プロペラ面を細かくする — cm 入力は不要）**
   翼厚は約 1 mm と薄いので，粗いと翼が解けず揚力が出ない．ただし**種類「自動」のメッシュは
   cm ではなく「サイズ調整」スライダー（相対値，小さいほど細かい）＋「自動サイズ細分割」の
   チェックで決める**（per-surface の cm 欄はこの版では実用的に出ない）．次の手順で行う：

   1.「メッシュサイズ」を開く（種類は「**自動**」のまま）．
   2.「自動サイズ細分割」の **☑ サーフェス細分割 を ON**（曲面・薄部を自動で細かくする）．
      薄翼がまだ解けない場合のみ **☑ ギャップ細分割 も ON**（薄板を貫く方向に要素を確保．要素数は増える）．
   3.「**サイズ調整**」スライダーで要素数を追い込む．**予想される要素数（ダイアログ左下）≈ 1〜数百万**を狙う．
      - 実績：**サイズ調整 0.4 ＋ サーフェス細分割 ON ≈ 92 万要素**（本件の妥当な初期値）．
      - **粗すぎ（例 20 万）は翼が再現されず NG**，**細かすぎ（数千万）は RAM 不足/数日で NG**（RAM ≈ 2 GB/100 万要素）．
   4.「**適用**」→ メッシュ生成 → **翼の断面を拡大し，翼が解けている（穴がない）か一度目視確認**．
   - ★ サイズ調整値とチェック内容は **3 ケースで同一**にする（公平比較）．まずは完走優先で，
     理想の「板厚 3 要素」に拘らず数百万で通す．3 ケース同一なら相対比較（A/B/C の優劣）は成立する．

## 6. 数値計算の実行（リボン「セットアップ」→「実行」／資料 2.4）

「実行」ダイアログ「コントロール」タブで，§0 の表に従い設定：

| 項目 | 値 |
|---|---|
| 解析モード | **非定常解析** |
| 時間刻み幅 | **0.003** s（1.8 deg/step） |
| **終了時刻（Stop Time）** | **`-1`**（時間では止めない．早期終了を防ぐ） |
| **内部反復係数** | **3**（資料は 1 だが発散したため増やす） |
| 保存間隔（結果） | **0.03**，「次に基づく」＝**秒** |
| **実行する時間ステップ数** | **400**（結果用＝0.003×400＝1.2 s＝2 回転）／安定テストは **200**（1 回転） |
| **次から継続（Continue from）** | **t0（最初／ステップ 0）** |
| ソルバーコンピュータ | MyComputer（コア数は 2ⁿ） |

> ⚠️ **dt とステップの関係**：dt を 0.003 にしたので **200 ステップ＝1 回転**．揚力平均には**2 回転＝400 ステップ**が要る．
> まず 200（1 回転）で発散しないか確認し，問題なければ 400（2 回転）で本番を回す．
>
> ⚠️ **「step 0 で即終了」を防ぐ要点**（§トラブルシュート参照）：
> **終了時刻＝`-1`**，**実行する時間ステップ数＝400（テストは 200）**，**次から継続＝t0**，そして
> 下記の**インテリジェント解析制御＝無効**．この 4 つが揃っていないと 0 ステップで「正常終了」する．

次に「**ソリューションコントロール**」を開き：

- **インテリジェント解析制御の「有効」が外れている**ことを確認．
- **流れ：圧縮性流体を選択（安定化）**．**全温度 ≈ 300 K（＝26.85℃）**，**伝熱計算＝オフ**，静水圧＝オフ．
  - ⚠️ 温度の**単位（ケルビン/摂氏）と値**を必ず確認：300 K＝26.85℃．`26.85` をケルビンにすると誤り（密度 ρ=p/RT が狂う）．
  - M≪0.3 なので**結果は非圧縮とほぼ同じ**だが，回転（運動）を伴う計算は圧縮性の方が安定（資料の助言）．
- 乱流モデル：**層流（Laminar）**．ソリューションコントロール →「乱流」→ ドロップダウンで **層流**．

> **流れ・乱流モデルの方針（安定性優先）**
> - 本件は **Re≈500 と非常に低く**，物理的に**ほぼ層流**．**層流＋圧縮性**が最も安定かつ妥当なので**まずこれで1本通す**．
> - **SST k-ω** は低 Re・剥離に強いが**ソース項が硬く発散しやすい**（本件で step 66 発散）．**安定確認後に追加で回す**．
> - **k-ε** は壁関数前提で薄翼の低 Re・剥離に弱く不適（既定でも使わない）．
> - ★ 流れ・乱流モデルとも **3 ケースで同一**にする（公平比較）．層流と SST の 2 本立てで
>   A→B→C の優劣順が一致するかを見るのが目的（§9）．

→「実行」．時間がかかるので**可視化しながら**進める．
出力バーに途中経過が出る．**「実行する時間ステップ数」が 400（テストは 200）になっていること**を必ず確認
（ここが 1〜0 だと一瞬で終わる）．最初の数ステップの所要時間も見て，**1 ステップ ≳1〜2 分なら重すぎ**
→ §5 項2 で外箱を粗く／項4 で翼面を緩めてやり直す．

> 中断・再開：「実行」をダブルクリック→「停止」で中断（アニメデータは自動保存）．
> 保存先 `…\<アセンブリ名>\設計 1\シナリオ 1\` の `.cfdst` から再開できる．
> 再開時は「実行する時間ステップ数」を残り回数に直す（例 400 のうち 21 済みなら 379）．

### トラブルシュート：非定常が「step 0 で正常終了」してしまう

ログが `解析開始` → すぐ `ポスト処理` → `解析が正常に終了しました` と流れ，**プロットが空
（開始 1／終了 0）＝1 ステップも進んでいない**場合．Autodesk CFD の既知挙動で，原因は次のいずれか：

1. **次から継続（Continue from）が前回の最終ステップを指している**（リスタート扱い）．既に終了時刻に
   達しているため**残りステップ＝0**．→ **次から継続を t0（最初）に戻す**。前回の結果はクリアして
   t0 から解き直すのが確実．
2. **終了時刻（Stop Time）と実行ステップ数は「先に満たした方」で停止**する．終了時刻が小さい/既達だと
   0 ステップで止まる．→ **終了時刻＝`-1`** にし，**実行する時間ステップ数＝400（テストは 200）** で止める．
3. **インテリジェント解析制御（ISC）が ON**．収束監視で「変化なし」と判断して停止する．
   → ソリューションコントロールで **ISC を無効**．

> 注：ログの `Intelligent Wall Formulation: ON` は SST k-ω の壁処理であり**別物（正常）**．ISC とは異なる．
> 参考：Autodesk「Transient analysis is stopping prior to specified stop time」，
> フォーラム「Transient Analysis - Time Step Problem」．

### トラブルシュート：途中で発散して「ソルバーが予期せぬ終了」

残差が途中まで平坦だったのに，あるステップ付近で Vx/Vy/Vz/圧力が急増 →
**「ソルバーが予期せぬ終了をしたため解析が停止しました」**（NaN クラッシュ）．数値不安定が原因．
資料 3.3 も「途中で再計算の場合，計算が発散する場合があった」と注記．対処（効果の大きい順）：

1. **dt を下げる（0.006 → 0.003 s＝1.8°/step）** ＝回転翼では最も効く．1 ステップの回転量・メッシュ歪みが半減する
   （資料も同じ 1.8°/step を「安定性重視」で採用）．まだ落ちるなら 0.002 へ．
2. **流れを圧縮性流体にする**．資料明記「運動を伴う場合は圧縮性が安定」．M≪0.3 で結果は非圧縮とほぼ同じ．
   全温度 ≈300 K，伝熱オフ（§6）．
3. **内部反復係数を増やす（3 → 4〜5，必要なら 10）**．各時間ステップを十分収束させて安定化．
4. **乱流モデルを層流（Laminar）にする**．Re≈500 では物理的に妥当で，SST k-ω のソース項の硬さを回避．

> ⚠️ **シリンダー細分化“領域”を足すと発散は悪化しやすい**：動くソリッドが細かいセルを 1 ステップで多く横切り，
> 局所クーラン数が上がるため．発散対策中は領域を足さない（足すなら dt も併せて下げる）．
> 発散したケースから**そのまま再開すると再発散しやすい**（資料の注記どおり）．上の安定化設定に変えて
> **次から継続＝t0** で**頭から解き直す**のが確実．発散前の値は揚力平均には使えない．

> 発散したケースから**そのまま再開すると再発散しやすい**（資料の注記どおり）．上の安定化設定に変えて

## 付録 A. 自動化メモ（CFD Python API）

Autodesk CFD には SWIG 製 Python API が同梱：`C:\Program Files\Autodesk\CFD 2027\Python\CFD\`
（`Setup.py`, `Results.py` ＋ `_*.pyd`）．雛形は [scripts/cfd_run.py](../scripts/cfd_run.py)．

- **実行環境の制約**：`.pyd` は CFD の **Python 3.13** 向け．外部 3.13 に `os.add_dll_directory(<CFD root>)`
  ＋ `sys.path` に `<CFD>\Python` を足せば `import CFD.Setup`/`CFD.Results` は通る（3.8 は ABI 非互換で不可）．
  ただし **`Setup.DesignStudy.Create()` を外部プロセスで呼ぶと 0xC0000005 で即クラッシュ**．API は
  アプリ文脈（DSE）必須で，**ヘッドレス自動化は不可**．→ **CFD 本体の Script Editor から実行**する．
- **取込は `createFrom()` ではなく `ds.createStudyFromAsmTranslator(step, name)`**（`createFrom` は
  `ValueError`）．`<case>_cfd.step` は **2 パーツ**（内側=プロペラ，外側=外箱）で入る．`part.volume()` は 0 を
  返すので識別は `part.boundingBox()` の大小（小=prop，大=box）．
- **自動化の限界**：`turbModel`／材料タイプ／`BoundaryCondition.pressureType` などの **enum が SWIG 非公開**
  （C++ 側のみ）．よって材料・回転・BC・乱流の設定は **GUI（本書 §2–§7）で行うのが確実**．
  取込までは API 確認済み，結果抽出 `Results.WallResults(force/torque)` も API 可（ただし本課題は §7 の CSV）．
