# Autodesk CFD 2027 操作手順（GUI・クリック単位）

Case A / B / C を**同一設定**で解析し，推力 \(F_z\) を読み出すための，Autodesk CFD 2027
（日本語UI）のクリック単位の操作手順．設定の根拠・対応表は
[scripts/cfd_setup.md](../scripts/cfd_setup.md) と [10_autodesk_cfd.md](10_autodesk_cfd.md)，
形状は [caseA.md](caseA.md) / [caseB.md](caseB.md) / [caseC.md](caseC.md) を参照．

> **比較の鉄則**：3 ケースで変えるのは**読み込む形状ファイルだけ**．材料・回転数・境界条件・
> メッシュ方針・ソルバ・乱流モデルはすべて同一にする．Z 軸まわりの回転なので**回転中心は
> 3 ケースとも `(0,0,0)` で良い**（Z 軸上のどの点でも Mz・回転は不変）．

---

## 0. 前提

- 入力形状：`geometry/<case>/<case>_cfd.step`（プロペラ＋回転円筒 r56 mm＋外箱の**入れ子3ソリッド**）．
  無ければ Windows/Inventor で `python scripts/cfd_domain_inventor.py` で再生成．
- 単位：STEP は mm だが CFD は cm 表示（「ジオメトリ (cm)」）．長さを入れる欄は cm に注意．
- ソルバは **CPU 専用**（GPU 不可）．コア数は実行設定で 2ⁿ を指定．

## 1. 形状の取り込み（スタディ作成）

1. CFD 起動 → **「開始および学習」または「設計の追加/更新」** から新規スタディ．
2. ジオメトリに `geometry\caseA\caseA_cfd.step` を選択して取り込む．
3. 取り込むと **3 つの入れ子ボリューム**ができる（左の「デザインスタディバー」→「材料」配下に
   3 パーツ）．名前と対応：
   - `...ソリッド3_U_ソリッド2_U_ソリッド1`（最内）＝**プロペラ**（固体）
   - `...ソリッド3_U_ソリッド2`（中間の殻）＝**回転流体（円筒）**
   - `...ソリッド3`（最外の殻）＝**静止流体（外箱）**
   - 見分けがつかない時は，左ツリーでパーツをクリックすると 3D ビューで該当部が
     ハイライトされる（翼形状＝プロペラ，細い円筒殻＝回転流体，最大＝外箱）．

## 2. 材料（リボン「セットアップ」→「材料」）

各ボリュームを選択 →「材料」→ 割り当て：

| ボリューム | 材料 |
|---|---|
| プロペラ（最内） | 任意の**固体**（例：ABS／アルミ．構造は無視するので何でも可） |
| 回転流体（中間円筒殻） | **空気（Air）** |
| 静止流体（外箱殻） | **空気（Air）** |

## 3. 回転領域（リボン「セットアップ」→「モーション」）

1. **回転流体（中間の円筒殻＝`...ソリッド3_U_ソリッド2`）** を選択．
   - ※ MRF の回転領域は「プロペラを取り囲む円筒状の流体」に与える．内部のプロペラ固体は
     その回転枠に含まれるので，**プロペラ自体に別途モーションを与える必要はない**．
2.「モーション」→ **回転運動（Rotating）**．
3. 回転軸：**+Z**（`(0,0,1)`），回転中心：`(0,0,0)`，回転速度：**100 rpm**．
4. 定常（MRF 相当）．

## 4. 境界条件（リボン「セットアップ」→「境界条件」）

1. リボン「選択」を**サーフェス**にし，**外箱の外側6面**を選択（直接選択で6面クリック，
   または箱の外周面をまとめて）．
2.「境界条件」→ **圧力（Pressure）= 0 Pa（ゲージ圧）**．
3. プロペラ・スピンドル壁は既定の **no-slip 壁**（指定不要）．

## 5. メッシュ（リボン「セットアップ」→「メッシュサイズ」）

1.「メッシュサイズ」→ **自動**．
2. **薄翼を解像するため翼面を細分化**：モデルメッシュ設定で「サーフェス細分割」を**有効**にし，
   プロペラ面のメッシュを細かく（薄翼1 mm に最低3要素目安）．回転円筒にも体積細分化．
   - ★ この細分化レベルは **3 ケースで同一**にする（公平比較）．
3. メッシュ生成．節点・要素数は出力バーで確認できる．

## 6. 実行（リボン「セットアップ」→「実行」）

実行ダイアログで以下を確認・設定：

- **流れの計算：オン**，**圧縮性：非圧縮性流体**，**伝熱計算：オフ**．
- **乱流：乱流，乱流モデル：SST k-omega**（出力に `Turbulence model: SST_k-omega` と出る）．
- **定常（Steady）**，反復回数：100〜（壁面力 \(F_z\) のモニタが平坦化するまで．収束プロットで確認）．
- ソルバーのコア数：**2ⁿ の最大**（GPU 設定は無い）．
- 実行 → 出力バーに「**解析が正常に終了しました**」が出れば完了．

## 7. 結果＝推力 \(F_z\)・トルク \(M_z\) の抽出（リボン「結果」タブ）

1. リボンを **「結果」** タブに切り替え．
2. **壁面計算機（Wall Calculator）** を開く（結果タブのレビュー系ツール）．
3. リボン「選択」を**サーフェス**にし，**プロペラ（とスピンドル）の壁面**を選択．
4. 壁面計算機で：
   - **力（Force）の Z 成分 = 推力 \(|F_z|\)** ［N］
   - トルク軸を **+Z**，軸点 `(0,0,0)` に設定 → **モーメント（Moment）の Z 成分 = トルク \(M_z\)** ［N·m］
5. 推力係数 \(C_T = |F_z|/(\rho n^2 D^4)\)，\(\rho=1.225,\ n=1.667\ \mathrm{rev/s},\ D=\)掃引径．

## 8. 3 ケース＋層流の繰り返し

1. **caseB / caseC** を §1〜§7 で同様に（`caseB_cfd.step` / `caseC_cfd.step`，**変えるのは形状のみ**）．
2. 低 Re（≈500）なので，各ケースを **乱流モデル＝層流（Laminar）** でも再計算し，A→B→C の
   優劣順が乱流モデルに依らないことを確認（§6 の乱流モデルを層流に変えて再実行）．

## 9. 結果の記録

得られた \(|F_z|\)（SST と層流）を [summary.md](summary.md) の比較表に記入し，
[caseC.md](caseC.md) の考察（実測 \(F_z\)・\(C_T\)，A/B 比，流れ場）に反映する．

> 補足（自動化）：本手順の取り込みまでは API（`scripts/cfd_run.py`，
> `ds.createStudyFromAsmTranslator(...)`）で確認済みだが，材料タイプ・乱流モデル・BC タイプの
> enum 値が SWIG ラッパに非公開のため，材料以降は本 GUI 手順で行うのが確実
> （[scripts/cfd_setup.md](../scripts/cfd_setup.md) §9）．
